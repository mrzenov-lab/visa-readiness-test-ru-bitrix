# Правила проекта для AI-агента

Этот файл хранит постоянные правила работы AI-агента внутри проекта. Перед внесением изменений агент должен учитывать эти инструкции как дизайн-бриф и как рабочие ограничения.

## Назначение

AGENTS.md нужен, чтобы агент понимал:

- как действовать при изменении интерфейса;
- когда задавать уточняющие вопросы;
- как выявлять ошибки и спорные решения;
- что нельзя менять без отдельного разрешения;
- какие визуальные правила проекта нужно сохранять.

## Работа с интерфейсом

- Использовать этот файл как дизайн-бриф для построения и доработки интерфейса.
- Избегать типичного AI-стиля: случайных градиентов, декоративных эффектов без смысла, перегруженных карточек и визуального шума.
- Держать отступы, размеры, цвета и состояния элементов в единой системе.
- Сохранять аккуратную, деловую, понятную визуальную подачу.
- Использовать уже существующие компоненты и паттерны проекта, если они есть.
- Если в проекте используется Tailwind CSS, работать через Tailwind-классы и не смешивать стили без необходимости.
- Проверять мобильную версию после заметных изменений в интерфейсе.
- Учитывать, что интерфейс должен быть удобен для кандидата, который может плохо читать по-русски или пользоваться страницей с телефона.

## Когда задавать вопросы

Агент должен задавать вопросы, если:

- изменение влияет на смысл продукта, юридические обещания или бизнес-логику;
- есть несколько равнозначных вариантов, и выбор зависит от позиции владельца проекта;
- не хватает данных для корректной реализации;
- предложенная правка может ухудшить пользовательский путь.

Если вопрос не критичный, агент должен принять разумное решение сам и коротко объяснить его.

## Проверка качества

После правок интерфейса агент должен проверить:

- читаемость текстов;
- отсутствие наложений и обрезанных элементов;
- работу на мобильной версии;
- понятность кнопок и основных действий;
- корректность ссылок и форм;
- сохранение общей визуальной системы.

## Ограничения

- Не менять бизнес-логику теста без явного запроса.
- Не удалять важные тексты, ссылки, формы и рекламные акценты без согласования.
- Не добавлять новые внешние сервисы, библиотеки или зависимости без необходимости.
- Не делать интерфейс похожим на шаблонную лендинг-страницу, если задача требует рабочий инструмент.
